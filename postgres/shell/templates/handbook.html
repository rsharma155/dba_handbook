01. DBA Principles

Collect evidence before changing anything on production PostgreSQL.

Never immediately: restart PostgreSQL, pg_terminate_backend all sessions, DROP INDEX, change shared_buffers without plan, or VACUUM FULL during peak.
14_Baselines/performance_snapshot.sql
Capture pg_stat_activity, wait events, and top queries before remediation.
04_Performance_Diagnostics/wait_events.sql
Note pg_postmaster_start_time() — many stats reset on restart.
SELECT pg_postmaster_start_time(), now();

14. Wait Events Knowledge Base

Common PostgreSQL wait events and fixes.

Lock / relation

Cause: Blocking DDL/DML
Fix: blocking_and_locks.sql

IO / DataFileRead

Cause: Disk reads, cold cache
Fix: disk_io_analysis.sql, indexes

WalSync

Cause: WAL flush latency
Fix: checkpoint_and_wal.sql

Client / ClientRead

Cause: App not consuming rows
Fix: connection_analysis.sql

LWLock / BufferContent

Cause: Hot page contention
Fix: Reduce concurrent updates

Timeout

Cause: statement_timeout fired
Fix: Optimize query or raise timeout
